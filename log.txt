Multi-Modal Evidence Review
Build a system that verifies damage claims using images, a short claim
conversation, user history, and minimum evidence requirements.
Each claim is about one of three object types: car, laptop, package.
Your system must decide whether the submitted images support the user's claim,
contradict it, or do not provide enough information.
The images are the primary source of truth. The user conversation defines what
needs to be checked. User history can add risk context, but should not override
clear visual evidence by itself.

What the system should do:
* extract the actual damage claim from the conversation
* inspect one or more submitted images
* decide whether the image evidence is sufficient
* identify the visible issue type
* identify the relevant object part
* decide whether the claim is supported, contradicted, or lacks enough information
* select the image IDs that support the decision
* flag image quality, mismatch, authenticity, or user-history risks
* estimate severity
* produce short justifications grounded in the images

Claims: A set of claims is provided. Some include expected output for
validation; others do not. Run all claims with your agent and submit the
results as a CSV.

Requirements:
* Must read the provided CSV files and local images.
* Must produce output.csv with the exact schema in problem_statement.md.
* Must include an evaluation workflow
* Must avoid hardcoded test labels or file-specific answers.

user_history.py:

evidence_requirements.py:

  full_reference_block() sends ALL applicable rules to the model (not just best-match)
  Soft keyword-overlap matching as fallback; DEFAULT_REQUIREMENT as final fallback

decision.py:
  build_final_row():
    1. Union model risk_flags + injection flag + missing-image flag + history flags
    2. Clamp all fields to allowed vocab
    3. Internal consistency guards:
       - not_enough_information → evidence_standard_met=false (claim_status drives)
       - not_enough_information → issue_type=unknown, severity=unknown
    4. manual_review_required policy (deterministic):
       - model_confidence=="low" OR
       - claim_status=="contradicted" OR
       - possible_manipulation/non_original_image flagged OR
       - text_instruction_present flagged OR
       - history risk AND claim_status != not_enough_information
    5. supporting_image_ids filtered to only actual submitted image IDs
    6. History note appended to justification text
  build_fallback_row(): used when model call fails after retries
